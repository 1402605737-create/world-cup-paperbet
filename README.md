# 世界杯纸上竞猜

**GitHub 仓库：** https://github.com/1402605737-create/world-cup-paperbet
**在线演示：** https://world-cup-paperbet-web.vercel.app
**后端健康检查：** https://world-cup-paperbet-api.vercel.app/health

世界杯纸上竞猜是一个移动端优先、由真实赔率驱动的虚拟策略学习实验室。用户可以选择真实世界杯胜平负赔率，使用练习币建立模拟单，再让多个不同人格、不同任务的人工智能角色共同审查判断。

> 合规边界：本产品只使用虚拟练习币进行模拟学习，不提供真实投注、资金充值、资金提取、支付或外部平台跳转。正式发布前仍需完成法律、年龄、地区与应用商店政策审查。

> 网络访问说明：部分网络会错误解析或关闭 Vercel 免费 `.vercel.app` 前端域名。若浏览器显示 `ERR_CONNECTION_CLOSED`，请通过本机代理 `http://127.0.0.1:7897` 访问。该环境已验证代理访问正式前端返回 HTTP 200；部署本身处于正常状态。

## 当前可用功能

- 从每场比赛进入统一整合下单中心，真实胜平负赔率与练习玩法集中选择
- 练习币可随意补充，补充额度不计入盈亏
- 正式盈亏、复盘盈亏与待结算风险敞口持续分账统计
- 单场胜平负票、快捷投入、固定赔率快照、预计返还与票号
- 每场比赛提供统一整合下单中心
- 整合支持胜平负、主队让 1 球胜平负、主队让 2 球胜平负、猜比分和总进球数
- 支持跨玩法多选、1/5/10/20 倍、分项预计返还与本场票据历史
- 已结束比赛支持胜平负、让球胜平负、猜比分和总进球模拟补购复盘票，并立即结算
- 进入虚拟模拟页自动结算可匹配的真实已结束比赛，也可手动立即结算
- 固定记录真实赔率快照、选择理由、虚拟投入和潜在返还
- 三个独立人工智能角色并行分析：数据分析师、反方审计员、风险管理员
- 主教练角色汇总分歧、给出裁决、虚拟仓位上限和复盘问题
- 主教练明确给出“虚拟买入 / 小仓试验 / 建议观望”、建议金额与执行条件
- 主教练建议执行时，Agent 使用独立虚拟账本自动出票；建议观望时不出票
- 用户与 Agent 的累计盈亏、命中率、已结算数量和待结算敞口并排对比
- 已结束比赛和已结算票据统一显示双方国旗与最终比分
- 会审历史、模拟单历史、经验、等级和纪律分永久保存在当前设备
- 中文人工智能学习助手，包含练习任务、参考答案和本机学习历史
- 真实世界杯赛程、赛果、赔率和国家旗帜
- 每日自动同步 2022 年世界杯完整历史赛果
- Supabase 独立数据库命名空间与最小权限应用角色
- 独立 Vercel 前端与后端项目
- 明确拒绝使用虚构赛程、赛果、赔率或收益承诺

当前供应商密钥已经配置。页面已验证可读取 2022 年世界杯 64 场完整赛果，以及本届世界杯当前真实赔率与未来赛程。

## 当前数据边界

- 已完成：2022 年世界杯完整赛果、球队国旗、每日自动同步。
- 已完成：本届世界杯当前真实赔率、未来赛程，以及实时/近三天比分接口。
- 尚未完成：更早四届世界杯赛果；当前免费套餐不提供这些届次。
- 已完成：进入虚拟模拟页时自动核对当前比分源并结算可匹配的已结束比赛，也可手动立即结算。
- 尚未完成：服务端持久化结算审计；当前票据、结算和盈亏历史保存在用户设备。

系统会诚实展示以上边界，不会为了填满页面而生成假数据。

胜平负使用供应商真实赔率。当前供应商没有提供让球、比分和总进球赔率，因此这些玩法使用醒目标注的虚拟练习赔率。当前比分源没有稳定提供半场比分，所以暂不提供无法可靠结算的半全场玩法。

## 本地运行

```bash
npm install
npm run build
npm run test
npm run dev:backend
npm run dev:frontend
```

本地运行时，可将 `.env.example` 复制为本地 `.env`。不得提交任何凭据。前端只读取 `EXPO_PUBLIC_API_BASE_URL`，所有数据源与人工智能密钥仅保存在后端。

## 人工智能服务

后端通过兼容接口调用 DeepSeek。所有自然语言输出均要求使用简体中文，输出必须通过结构校验。非法 JSON 或违反内容安全要求的结果会重试一次；仍失败时会明确返回错误，不使用虚构替代结果。

主要接口：

- `POST /api/ai/learn`：生成中文学习讲解
- `POST /api/ai/strategy-panel`：并行运行三个独立角色，再由主教练汇总裁决
- `POST /api/ai/verify`：验证真实人工智能调用
- `GET /api/system/config-status`：读取真实数据源配置状态
- `GET /api/matches`：从 API-Football 读取真实世界杯赛程与赛果
- `GET /api/odds`：从 The Odds API 读取真实世界杯胜平负赔率
- `GET /api/current-scores`：读取本届实时与近三天比分
- `GET /health`：读取后端、数据库和人工智能健康状态

## 真实数据原则

当前版本已实现真实数据供应商调用：

- `API_FOOTBALL`：通过 `https://v3.football.api-sports.io/fixtures` 获取真实赛程与赛果
- `THE_ODDS_API`：通过 `soccer_fifa_world_cup` 获取真实世界杯胜平负赔率

没有真实密钥时，界面会明确提示数据源待配置，不会生成假赛程、假赛果、假赔率或假策略。

所需后端环境变量：

```text
SPORTS_API_KEY=
SPORTS_API_BASE_URL=https://v3.football.api-sports.io
SPORTS_LEAGUE_ID=1
SPORTS_SEASON=2022
CRON_SECRET=
ODDS_API_KEY=
ODDS_API_BASE_URL=https://api.the-odds-api.com
ODDS_SPORT_KEY=soccer_fifa_world_cup
ODDS_REGION=eu
ODDS_MARKET=h2h
ODDS_FORMAT=decimal
```

## 数据库隔离

部署复用现有 Supabase Free 项目，但不会触碰其他应用：

- 独立数据库命名空间：`world_cup_paperbet`
- 独立登录角色：`world_cup_paperbet_app`
- 事务连接池端口：`6543`
- 所有生产查询显式使用 `world_cup_paperbet.表名`
- 应用角色不能执行数据库结构操作，也不拥有业务表
- 行级安全策略仅面向 `world_cup_paperbet_app`

生产启动时不会创建、修改、删除或迁移数据库对象。

Vercel 定时任务每天调用一次受 `CRON_SECRET` 保护的 `/api/cron/sync-results`，将 API-Football 返回的 2022 年世界杯历史赛果和国家旗帜更新到当前项目独立数据库命名空间。API-Football 免费套餐当前不允许访问本届赛果，因此本届当前赔率与未来赛程由 The Odds API 单独展示。
